HTTP

  • Hypertext Transfer Protocol (HTTP)

    • Protocolo de aplicación mayormente usado en la actualidad, siendo la base de lo que conocemos como internet.
    • Su propósito es la adecuación del lenguaje HTML y otros documentos web a través del internet.
    • Una pàgina web esta compuesta de objetos. Estos objetos pueden ser archivos HTML, imagenes, Java Applet, audio, video, etc.

  • Uniform Resource Locator (URL)

    • Es una cadena de caracteres asignada a una direcciòn ùnica de los recursos de uan aplicaciòn web o documento de internet.
    • Formato:
    • esquema://dominio/directorio/archivo

  • Aplicación HTTP

    • El HTTP maneja el modelo Cliente/Servidor donde:
    • Cliente: es el browser que requiere, recibe o despliega contenido web.
    • Servidor: Servidor Web que envia objetos en respuesta a requerimientos.

  • Conexiones HTTP

    • HTTP No-Persistente: Solo puede ser enviado un objeto en una conexión.
    • HTTP Persistente: Multiples objetos pueden ser enviados en una misma conexión entre el cliente y el Web Server.

  • Cookies

    • Es un archivo que se guarda en el dispositivo del usuario y le permite al Web Server distinguir a ese usuario cuando cambia de página web.
    • Guardan preferencias de usuario.

FTP

  • File Transfer Protocol (FTP)

    • Es el protocolo de aplicación que permite la transferencia de información o archivos entre computadoras.
    • Utiliza dos puertos, uno para el control de la transferencia y otro para la manipulación de los archivos a enviar.
    • Puerto utilizado: 20 y 21 / TCP

    • FTP modo activo: El cliente se conecta desde un puerto aleatorio hacia el puerto 21 del servidor y le indica que envie las respuestas por el puerto n+1 del cliente.
    • FTP modo pasivo: El cliente se conecta desde dos puertos aleatorios hacia el 21 del servidor desde el primero y el servidor envia un puerto por el cual enviar las respuestas.

SMTP

  • Protocolo de red basado en texto, utilizado para el intercambio de mensajes de correo electrónico entre computadoras u otros dispositivos.
  • En el SMTP los clientes y el servidor cuentan con 2 componentes importantes:
  • User Agents: Prepara el mensaje y lo ‘empaqueta’ para su transmisión.
  • Mail Transfer Agent: Transfiere el correo a través de Internet.

  • Mail Gateway

    • Un gateway orientado a correo electrónico, está diseñado para conectar dos protocolos de correo diferentes.
    • Ejemplo: Dos sistemas de correo SMTP y POP3

  • Limitaciones

    • Solo usa formato NVT 7 bit ASCII.
    • No tiene mecanismos de autenticación.
    • Los mensajes no se encriptan.
    • Susceptible al mal uso (spam).
