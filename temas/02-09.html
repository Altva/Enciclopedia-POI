IPv6

  • Problemas con IPv4:

    • - Espacio IPv4 limitado y mal distribuido (EEUU vs. Resto del mundo)
    • - Encabezado de tamaño variable y no óptimo
    • - No muy flexible a extensiones y opciones
    • - Difícil implementar QoS
    • - Autentificación y privacidad no incluidos
  • En 1992, la IETF comenzó a buscar el protocolo de nueva generación para el Internet.
  • Dos años después se adoptó el Simple Internet Protocol Plus (SIPP).
  • En 1995 se llamó Internet Protocol versión 6 (IPv6).
  • Una dirección IPv6 se muestra de la forma:
  • xxxx.xxxx.xxxx.xxxx.xxxx.xxxx.xxxx.xxxx

  • Caracteristicas del IPv6:

    • - Nuevo formato de cabecera, mayor espacio de dirección.
    • - Direccionamiento eficiente y jerárquico e infraestructura de ruteo.
    • - Seguridad incluida.
    • - Soporte para multicasting.
    • - Soporte para auto configuración.
    • - Mejor soporte para calidad del servicio (QoS).
    • - Compatibilidad con el IPv4.
  • La cabecera del IPv6 se simplifica eliminando campos innecesarios. Permite procesamiento mas eficiente.
  • En el IPv4, la cabecera tiene un tamaño variable de 20 bytes, mientras que en el IPv6, tiene un tamaño fijo de 40 bytes.
  • IPv6 usa el modelo de direccionamiento de 128 bits, usando mas espacio que las direcciones IPv4 de 32 bits.

  • Una dirección IPv6 tiene 3 formas de representación:

    • - Forma colon-hexadecimal:

      • Por ejemplo una dirección IPv4 en decimal se vería: 26312.423.256.61020.65535.55234.6123.56432
      • La representación IPv6 en formato colon-hexadecimal: 66CB:01A7:0100:EE5C:FFFF:D7C2:12EB:DC70

    • - Forma comprimida:

      • Se pueden omitir los ceros (0’s). Con la dirección: 80FF:10FE:0E00:00FF:0000:0000:0000:0001
      • Comprimida sería: 80FF:10FF:E00:FF:0:0:0:1
      • Para reducir aún mas el tamaño, se pueden comprimir los grupos de 0’s de 16 bits en :: 80FF:10FE:E00:FF::1
      • NOTA: El símbolo :: solo aparece 1 vez en una dirección.

    • - Forma mixta:

      • Consiste en una combinación de una dirección IPv6 y una IPv4 en el formato: x:x:x:x:x:x:d:d:d:d
      • Donde las x representan lo valores hexadecimal de los elementos mas a la izquierda de la dirección IPv6, y las d representan el valor decimal de la dirección IPv4.

  • Tipos de direcciones IPv6

    • - Unicast: Existen diversas formas dependiendo del alcance:
    • - Enlace local: No pasan a través de un router. Estas direcciones se usan cuando el equipo se enciende y aún no ha adquirido sus direcciones.
    • - Sitio local: Se utilizan en el interior de una organización. Los prefijos son FECx, FEDx, FEEx, FEFx. //IPv4 privadas
    • - Global: Son direcciones públicas utilizadas por el Internet con IPv6. //IPv4 publicas
    • - Multicast: Una dirección multicast es un identificador para un grupo de direcciones y es usada para broadcasting.
    • - Anycast: Una dirección anycast se asigna a mas de una interfaz. Los paquetes enviados a una dirección anycast se rutean a las interfaces mas cercanas.
  • El anycast es una característica del IPv6 y no está disponible en el IPv4.
