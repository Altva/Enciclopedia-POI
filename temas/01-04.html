Internet y DHCP

  • Hardware address

    • Dentro de cada paquete de información hay una cabecera que contiene información de la dirección
    • Esta dirección esta grabada en forma definitiva en la tarjeta de Red por el fabricante.
    • La dirección es única entre todos los manufactureros. El dato es de 12 dígitos hexadecimales.Donde los primeros 6, se le asignan al fabricante, y los últimos 6 los asigna el fabricante. Ej. 00:BA:D6:0E:65:A4

  • Broadcast Packet

    • Cuando la comunicación es entre dos hosts se agrega a la cabecera la dirección de origen y destino. Cuando se quiere transmitir un mensaje global, se le asigna:
    • FF:FF:FF:FF:FF:FF.

  • Capas de internet

    • La capa de internet del modelo TCP/IP está entre la capa de transporte y la de enlace. Sus protocolos son responsables de direccionar y rutear paquetes.Ejemplos:
    • Internet Protocol (IP)
    • Adress Resolution Protocol (ARP)
    • Internet Control Message Protocol (ICMP)
    • Internet Group Message Protocol(IGMP)

  • Internet Protocol(IP)

    • Una porción de la IPO define en que red esta el host, a través de una máscara de subred o Subnet Mask. Cuando se va a enviar un paquete por la red, el protocolo IP necesita saber si el destino es de una red local o a una red remota, esto lo determina con la máscara de subred.

  • Default gateway

    • Cuando el protocolo IP necesita enviar datos fuera de la red local, lo tiene que hacer mediante el default gateway(router). El default gateway, es la dirección del host en la red que ofrece la salida de la red hacia otras. Este host debe de tener 2 tarjetas de red, una para la red local, y otra para conectarse con las redes remotas.

  • Routing table

    • Es una tabla que contiene direcciones indicando la mejor ruta hacia otras redes.
    • Cuando la red es local, el IP busca la dirección del host de destino; y cuando la red es remota, el IP busca en su tabla de ruteo,por una ruta explícita,que en caso de no hallarla, utiliza nuestra default gateway.

  • Adress Resolution Protocol

    • ARP es un protocolo que resuelve una dirección IP en una dirección física.
    • ARP, primero intenta resolver las direcciones a través del ARP chache, que es una área de memoria que mantiene las relaciones de direcciones ya resueltas. En caso de no encontrarse la dirección física en el ARP chache, entonces el protocolo hace un ARP request.
    • La información en el ARP chace, dura sólo dos minutos. Si hay una transacción y se confirma la misma relación, dura otros dos minutos, por si las direcciones cambian.
    • En la tabla ARP cache, la primer columna es la IP, la segunda MAC, y la tercera es la manera en la que se dio de alta: Dynamic es porque se hizo a través de un ARP request, y Static, si se hizo manualmente

  • Internet Control Message Protocol (ICMP)

    • Es un protocolo utilizado para enviar mensajes de error, ejecutar diagnósticos y controlar el flujo de datos. Por ejemplo, cuando el ICMP ejecuta una utilería llamada piing, para ver el eco de la comunicación

  • Internet Group Management Protocol (IGMP)

    • Es un protocolo que habilita a un host enviar un stream de datos a varios hosts al mismo tiempo. El stream es una serie de paquetes enviados sin esperar reconocimiento de llegada.
    • La dirección de llegada del IGMP es llamada multicast adress y es reservada para este propósito.
    • El audio y video se transmiten desde este protocolo. Por ejemplo: Twitch

  • Capa de transporte

    • La capa de transporte del TCP/IP está formada por el protocolo TCP y UDP
    • EL TCP provee una conexión orientada y confiable. Es lento y utilizado para grandes montos de información.
    • El UDP no provee conexión y si una conexión poco confiable. Es rápido, pero para pequeñas cantidades de información.

  • Capa de aplicación

    • Es la parte del TCP/IP donde se realizan las solicitudes por datos y servicios. Las aplicaciones de esta capa están esperando por las solicitudes escuchando a sus respectivos puertos.
    • Un puerto es el número que se utiliza para enviar o recibir direcciones. Este distingue a la aplicación en comunicación.
    • Los protocolos TCP y UDP manejan 65536 direcciones distintas.
    • Un socket es la combinación de la IP, protocolo y puerto.
    • De esta manera una aplicación distingue la ruta por la que hará comunicación. Ejemplo: 192.168.2.0.TCP.80

  • Puertos conocidos

    • La IANA (Internet Assigned Number Authority) ha reservado algunos puertos:
    • TCP:80 HTTP
    • TCP:21 FTP
    • TCP:20 FTP-Data(TFTP)
    • UDP:161 Simple Network Managment Protocol (SNMP)

  • DHCP

    • El DHCP(Dynamic Host Configuration Protocol
    • El proceso del DHCP consta de 4 pasos principales:
    • Descubrimiento
    • Ofrecimiento
    • Solicitud
    • Reconocimiento
