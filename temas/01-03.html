Internet y DHCP

  • Hardware address

    • Dentro de cada paquete de información hay una cabecera que contiene información de la dirección
    • Esta dirección esta grabada en forma definitiva en la tarjeta de Red por el fabricante.

  • Capa de internet

    • La capa de internet del modelo TCP/IP esta entre la capa de transporte y la de enlace.

      • Internet Protocol

        • Este protocolo se encarga de determina la dirección de origen y destino de cada paquete.
        • El administrador de la red se encarga de asignar una direccion OP a la red en carácter de única.
        • Esta direccion se llama dirección lógica.

        • Ejemplo: 192.168.2.51

        • Cuando se va a enviar un paquete por la red, necesita saber si el destino es una red local o a una red remota, esto lo determina la máscara de subred

      • Default gateway

        • También llamado router es la dirección del host dek host en la red que ofrece la salida de la red hacia otras.
        • Este se utiliza cuando se quiere enviar algo fuera de la red local.

      • Routing table

        • Es una tabla que contiene direcciones indicando la mejor ruta hacia otras redes.
        • Cuando la red es remota, la IP se busca en su Routing table.

      • Address Resolution Protocol

        • ARP es un protocolo que resuelve una dirección IP en una dirección física.
        • Cuando la red es remota, la IP se busca en su Routing table.

      • Internet Control Message Protocol (ICMP)

        • Es un protocolo utilizado principalmente para enviar mensajes de error, ejecutar diagnósticos y controlar el flujo de datos.

      • Internet Group Management Protocol (IGMP)

        • Es un protocolo que habilita a un host a enviar un stream de datos a varios hosts al mismo tiempo
        • El audio y el video se transmiten utilizando este protocolo.

  • Capa de transporte

    • La capa de transporte del TCP/IP está formada solo por dos protocolos, el TCP y el UDP.
    • El TCP provee una conexión orientada y confiable. Es lento y utilizado para grandes montos de información y pretende asegurar que no se tendrá que reenviar.
    • El UDP no provee conexión y si una comunicación poco confiable. Es rápido y solo para cantidades pequeñas de información.
