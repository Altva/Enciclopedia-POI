TCP vs UDP

  • El protocolo HTTP es un protocolo cliente/servidor a través del cual dos sistemas pueden comunicarse, usualmente sobre una conexión TCP/IP.
  • Cabecera HTTP: Un mensaje HTTP consiste en una línea de comienzo, seguida por un grupo de headers, una línea vacía y algo de información.

  • Los headers HTTP pueden dividirse en tres categorías:

    • - Headers que se envían en la petición.
    • - Headers que se envían en la respuesta.
    • - Headers que se pueden enviar en petición y respuesta.
  • Peticiones HTTP: Cada cliente crea una petición y el servidor responde a esa petición.

  • Cada petición y respuesta tiene 3 partes:

    • - La línea de petición o respuesta.
    • - Una sección de cabeceras.
    • - El contenido que se envía con el mensaje.

  • Las líneas de petición del cliente se dividen en 3 secciones:

    • - Método HTTP.
    • - URI.
    • - Número de versión del HTTP.
  • Ejemplo: GET /default.aspx HTTP/1.1
  • Metodos HTTP: Un método es un comando HTTP que comienza la primer línea de una petición de un cliente. Informa al servidor del propósito de la petición del cliente.

  • Existen 7 métodos definidos para el HTTP:

    • GET - La info se envia por el URL
    • HEAD - Tiene un funcionamiento similar al GET, excepto que el servidor no envía nada en la porción de datos de la respuesta.
    • POST - Los datos enviados se encuentran en el contenido del mensaje de la petición del cliente.
    • DELETE - Realiza la petición para que el archivo sea borrado del servidor.
    • PUT - Realiza la petición para que el documento incluido en el cuerpo de la petición se cargue al servidor y esté disponible con la URI especificada.
    • TRACE - Utilizado para debuguear la cadena petición/respuesta.
    • OPTIONS - Realiza la petición por información sobre soporte del HTTP en el servidor web.
  • URI: Después del método de petición viene un URI especificando el archivo que se quiere obtener, cargar, etc.
  • Respuesta HTTP: La respuesta se divide en tres partes:
    • - Línea de respuesta del servidor.
    • - Información de cabeceras.
    • - Contenido del mensaje.
  • Después de que se envía la respuesta, el servidor:
  • HTTP/1.0: termina la conexión, al menos que:
  • Connection: Keep-Alive
  • HTTP/1.1: mantiene la conexión, al menos que:
  • Connection: close
  • Codigos de Estado: El código de estado es un número de 3 dígitos el resultado del servidora la petición del cliente.
