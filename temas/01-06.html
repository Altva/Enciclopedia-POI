TelNet

    • Desde los primero intentos de comunicación entre redes uno de los principales problemas que aquejaban a los científicos y que necesitaban resolver era el manejo remoto de un dispositivo desde otro.
    • El protocolo Telnet, que creció a la par del TCP/IP, fue diseñado para cubrir aquellos requerimientos.
    • En la actualidad la tecnología usada para este tipo de tareas se basa en principios del TelNet.Por ejemplo: el envío de correos, manipulación y transferencia de archivos a dispositivos, así como la carga de un sitio web.
    • Funcion principal: Definir un medio por el cual el usuario/proceso en un dispositivo pueda acceder y utilizar otra máquina como si estuviera conectado localmente. Maneja un modelo cliente servidor.
    • Un cliente TelNet, es un softare con el que interactúa el usuario a través de una interfaz. Este procesa la información obtenida a través del teclado para poder mostrarla en la máquina remota.Se ejecuta en un equipo remoto para permitir sesiones remotas.

    • Comandos

    • La mayor parte de la información que el usuario ingresa en la terminal toma forma de datos o comandos que son enviados hacia la aplicación usada. Debido a la falta de uniformidad en los datos, se tiene que definir una forma en que van a ser representados y así tomados en cuenta como un comando.

    • Traducción

    • TelNet incluye un número de códigos especiales para permitir al usuario controlar el funcionamiento del dispositivo remoto. También define un conjunto de comandos que son específicos para el propio protocolo Telnet. Éstos permiten que el cliente Telnet y el software del servidor Telnet se comuniquen. Estos son llamados "comandos del protocolo TelNet".
